This site documents the **architecture and internals** of `unitycatalog-rs` — a
Rust implementation of the Unity Catalog REST API, with Rust, Python, and
TypeScript client libraries.

User-facing tutorials, client how-to guides, and REST/configuration reference
now live in the consolidated open-lakehouse documentation. The pages here focus
on how the server is put together: service composition, the graph data model,
the code-generation pipeline, authorization, and the core trait contracts.

## Where to start

- **[Service composition](/explanation/service-composition/)** — how `ServerHandler` wires policy, storage, and secrets together.
- **[The graph data model](/explanation/graph-data-model/)** — resources as objects and associations.
- **[Code generation philosophy](/explanation/codegen-philosophy/)** — proto as the source of truth.
- **[Authorization architecture](/explanation/authorization/)** — principals, permissions, and decisions.
- **[Add a new resource type](/guides/add-resource-type/)** — the end-to-end contributor recipe.

## The open lakehouse ecosystem

`unitycatalog-rs` is one project in a broader ecosystem of open source tools for
building lakehouse data systems. While there is no official affiliation, these
projects are commonly considered part of it.

### Open table formats

- [delta](https://github.com/delta-io/delta)
- [delta-rs](https://github.com/delta-io/delta-rs)
- [iceberg](https://github.com/apache/iceberg)
- [iceberg-rust](https://github.com/apache/iceberg-rust)

### Catalogs

- [lakekeeper](https://github.com/lakekeeper/lakekeeper)
- [polaris](https://github.com/apache/polaris)
- [unitycatalog](https://github.com/unitycatalog/unitycatalog)

### Query engines

- [datafusion](https://github.com/apache/arrow-datafusion)
- [polars](https://github.com/pola-rs/polars)
- [duckdb](https://github.com/duckdb/duckdb)
